The Problem

Manual triage is a bottleneck

Read & understand

Parse issue title, body, comments. Figure out what the user actually wants.

Cross-reference

Check ADO, Ideas Portal, community forums, existing PRs. Is this a duplicate? Already planned?

Assess & decide

Estimate complexity, value, effort. Set priority. Assign to a team. Repeat hundreds of times.

15-30 min per issue × hundreds of issues = PMs drowning in triage

Design Principles

AI First, not AI only

🔍 LLM as judge, not source
Real APIs provide the data. The LLM evaluates relevance and explains why each result matters.
⚡ Parallel by design
8 enrichment sources + 2 analysis calls run concurrently. 60s total, not 60s × 10.
🛡️ Graceful degradation
Missing API keys or failed searches don't block triage. The agent works with what it has.
🧑‍💻 Human in the loop
The agent recommends. The PM decides. Every assessment includes rationale and confidence level.
