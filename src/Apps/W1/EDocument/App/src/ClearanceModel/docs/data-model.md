# Data model

The ClearanceModel data model consists of one temporary buffer table and four table extensions that add QR code storage to posted documents.

## EDocQRBuffer table

**Purpose:** Temporary data transfer object between source documents and QR viewer page.

```al
table 6196 "EDoc QR Buffer"
{
    TableType = Temporary;

    fields
    {
        field(1; "Document Type"; Text[30]) { }
        field(2; "Document No."; Code[20]) { }
        field(3; "QR Code Base64"; Blob)
        {
            DataClassification = CustomerContent;
        }
        field(4; "QR Code Image"; Media)
        {
            DataClassification = CustomerContent;
        }
    }

    keys
    {
        key(Key1; "Document Type", "Document No.")
        {
            Clustered = true;
        }
    }
}
```

**TableType = Temporary:** Never persisted to database; exists only in memory during QR viewer session.

**Field purposes:**

| Field | Type | Purpose |
|-------|------|---------|
| Document Type | Text[30] | Display name (e.g., "Sales Invoice") for viewer caption |
| Document No. | Code[20] | Document identifier for display and filename generation |
| QR Code Base64 | Blob | Source Base64-encoded PNG received from clearance service |
| QR Code Image | Media | Binary PNG converted for page control rendering |

**Why both blob types?**
- QR Code Base64: Storage format (matches API response from clearance services)
- QR Code Image: Display format (page controls require Media type)

**Key structure:** Composite key on Document Type + No. allows buffer to hold multiple QR codes if needed (though typical usage is single record).

## Table extensions

Four table extensions add QR code storage to posted documents:

### Posted Sales Invoice extension

```al
tableextension 6198 "Posted Sales Invoice with QR" extends "Sales Invoice Header"
{
    fields
    {
        field(6198; "QR Code Base64"; Blob)
        {
            Caption = 'QR Code Base64';
            DataClassification = CustomerContent;
        }
    }
}
```

**Field number 6198:** Matches object ID to avoid conflicts (convention: use extension ID as field number when possible).

**Blob type:** No subtype specified; stores raw binary or text data (in this case, Base64 text).

**DataClassification:** CustomerContent requires GDPR consideration (QR codes contain invoice amounts, participant IDs).

### Posted Sales Cr.Memo extension

```al
tableextension 6199 "Posted Sales Crd Memo with QR" extends "Sales Cr.Memo Header"
{
    fields
    {
        field(6199; "QR Code Base64"; Blob) { }
    }
}
```

**Same pattern:** All extensions use identical field definition for consistency.

## QR code data structure

The Base64 blob typically encodes a PNG image generated by the clearance service. The PNG contains a QR code matrix with embedded data following a jurisdiction-specific format.

**Example: Saudi Arabia ZATCA TLV format**

QR code payload (before Base64 encoding) uses Tag-Length-Value structure:
- Tag 01: Seller Name (UTF-8 encoded)
- Tag 02: VAT Registration No.
- Tag 03: Timestamp (ISO 8601)
- Tag 04: Invoice Total (with VAT)
- Tag 05: VAT Amount
- Tag 06: Invoice Hash (SHA-256)
- Tag 07: Signature (cryptographic signature)
- Tag 08: Public Key

Each tag is 1 byte, length is 1-2 bytes, value is variable length. The entire TLV structure is converted to QR code matrix, rendered as PNG, then Base64-encoded.

**Decoding workflow:**
1. Read Base64 text from blob
2. Decode to binary PNG
3. Parse QR code matrix to extract TLV bytes
4. Parse TLV structure to extract field values
5. Validate signature using public key
6. Call authority API with invoice hash to confirm clearance

## Storage size considerations

**PNG size:** Typical QR code PNG is 5-15 KB depending on:
- Data complexity (more data → denser QR matrix)
- Image resolution (300 DPI for print vs 72 DPI for screen)
- Error correction level (L=7%, M=15%, Q=25%, H=30% redundancy)

**Base64 overhead:** Base64 encoding increases size by ~33% (10 KB binary → 13.3 KB text).

**Database impact:** 1000 invoices with QR codes = ~15 MB blob storage.

**Optimization:** Consider storing only clearance ID + hash instead of full QR code, regenerating QR on-demand from stored values.

## Field metadata

**Caption:** "QR Code Base64" (user-visible name in page personalization).

**DataClassification:** CustomerContent triggers:
- Encryption at rest (if database encryption enabled)
- Export in GDPR data subject requests
- Exclusion from diagnostics telemetry

**No FlowField/CalcFormula:** Blob fields cannot be calculated; must be explicitly populated via code.

**No TableRelation:** Blobs don't support referential integrity (no FK constraints).

## Page control binding

The QR Code Image field binds to page controls:

```al
page 6198 "E-Document QR Viewer"
{
    SourceTable = "EDoc QR Buffer";

    layout
    {
        area(content)
        {
            field("QR Code Image"; Rec."QR Code Image")
            {
                ApplicationArea = All;
                Caption = 'QR Code';
                ToolTip = 'Scan this QR code to verify invoice clearance';
            }
        }
    }

    trigger OnAfterGetCurrRecord()
    begin
        EDocumentQRCodeManagement.SetQRCodeImageFromBase64(Rec);
    end;
}
```

**OnAfterGetCurrRecord:** Converts Base64 → Image each time record is displayed (ensures image is always in sync with blob).

**Media field rendering:** BC automatically scales image to fit control dimensions while maintaining aspect ratio.

## Extension field lifecycle

**Population:** Clearance service API callback writes to field:

```al
SalesInvoiceHeader.Get(InvoiceNo);
SalesInvoiceHeader."QR Code Base64".CreateOutStream(OutStream, TextEncoding::UTF8);
OutStream.WriteText(Base64Response);  // Write Base64 text from API
SalesInvoiceHeader.Modify();
```

**Consumption:** Users view via page action, export to file, or print on report.

**Deletion:** QR codes are deleted when parent document is deleted (standard cascade behavior for extension fields).

**Modification:** Typically immutable once set (clearance is final). No UI for editing QR codes.
