Redesign Review — Extending W1 for UK and AU/NZ

Context — Why Redesign?

  • Payment Practices W1 app — must support UK and AU/NZ regulations
  • UK — dispute tracking, construction retention, CSV export to gov portal
  • AU/NZ — small business supplier filtering (separate workstream)
  • Challenge — extend cleanly, no regression for W1/FR users

Current Architecture

Pipeline: Ledger Entries → Aggregated Lines → Header Totals. Two pluggable enums control data source and grouping. Payment periods are a single flat table shared by all headers.

The Third Dimension — Reporting Scheme

  • New enum: Reporting Scheme with handler interfaces
  • Standard — W1/FR, does nothing extra (zero regression)
  • Dispute & Retention — GB-specific logic
  • Small Business — AU/NZ filtering
  • Extensible enum — partners add their own schemes

Data Model — Old vs New

Aspect Old New
Payment Periods Flat table, one global set Template pattern: named templates per scheme
Reporting Scheme None Auto-detected from environment, user can override
GB Fields None ~40 qualitative fields in new separate table
AU Filtering None Small Business Supplier flag → non-small skipped
CSV Export None GB gov format (52 columns)
Key decision: GB qualitative fields live in a separate table — ~40 fields with complex visibility, "copy from previous period" feature, and conditional sections. Keeps the main card clean.

Payment Periods — Old vs New

OLD — Flat Table

  • One global set of periods
  • All headers share the same periods
  • Country defaults hardcoded in install
  • No way to customize per-header

NEW — Template Pattern

  • Named templates per scheme
  • Each header picks its own template
  • Users can create custom templates
  • One default template per scheme

GB — Dispute & Retention

  • Separate detail table for ~40 GB qualitative fields
  • Organized in groups: policies, contracts, payment terms, retention, disputes
  • "Copy from Previous Period" — carries over standing policies, clears period-specific data
  • CSV export — 52 columns matching UK gov portal format
  • SCF Payment Date — auto-populated from ledger, user can override
