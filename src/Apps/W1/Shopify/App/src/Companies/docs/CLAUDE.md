# Companies

Companies represent Shopify's B2B model, where a Shopify Company maps to a BC Customer record. This module handles the bidirectional sync of companies and their locations, including mapping strategies that determine how incoming Shopify companies are matched to existing BC customers. It is a Shopify Plus feature -- stores without Plus cannot use B2B companies.

## How it works

The sync lifecycle in `ShpfySyncCompanies.Codeunit.al` runs in three phases. First, if `Company Import From Shopify` is `AllCompanies`, it retrieves all company IDs from Shopify using paginated GraphQL (`ShpfyCompanyAPI.Codeunit.al`'s `RetrieveShopifyCompanyIds`), compares `Updated At` timestamps, and queues changed companies for import. If the setting is `WithOrderImport`, it only updates already-known companies (no new creation). Second, each queued company is processed by `ShpfyCompanyImport.Codeunit.al`, which retrieves the full company from Shopify, updates all locations, and then runs the mapping logic to find or create a BC customer. Third, if `Can Update Shopify Companies` is enabled, the export phase runs `ShpfyCompanyExport.Codeunit.al` against BC customers, sending changes back to Shopify.

Mapping is polymorphic via the `Shpfy Company Mapping` enum, which implements `ICompany Mapping`. Three strategies exist: `ShpfyCompByEmailPhone.Codeunit.al` matches by the main contact's email or phone against BC customer records; `ShpfyCompByTaxId.Codeunit.al` matches by the company location's tax registration ID against BC customers using a pluggable `Tax Registration Id Mapping` interface; `ShpfyCompByDefaultComp.Codeunit.al` always maps to the shop's configured default company. The mapping layer has a subtle two-interface design: `ICompanyMapping` has `DoMapping` (used from order processing, returns a customer number), while `IFindCompanyMapping` extends it with `FindMapping` (used during company import, returns boolean and mutates the company record). The `ShpfyCompanyMapping.Codeunit.al` orchestrator checks at runtime whether the chosen strategy implements `IFindCompanyMapping` and falls back to the email/phone approach if not.

Company locations are first-class entities stored in `ShpfyCompanyLocation.Table.al`. Each location tracks its own address, tax registration ID, payment terms, and -- crucially -- separate `Sell-to Customer No.` and `Bill-to Customer No.` fields. This means different locations within the same Shopify company can map to different BC customers for invoicing purposes. The export path handles this by iterating all locations for a company and updating each one individually based on the `Customer Id` GUID linking it to the originating BC customer. Payment terms sync is at the location level via `UpdateCompanyLocationPaymentTerms`.

## Things to know

- Companies map to BC **Customers**, not BC Contacts. The `Customer SystemId` on the company is a GUID pointing to a Customer record. The `Main Contact Customer Id` is the Shopify Customer ID (the person), while `Main Contact Id` is the Shopify Company Contact ID -- these are different Shopify entities.
- When exporting a new company, the code first creates a Shopify Customer (via the customer export pipeline), then creates the Company with that customer as the main contact. If the BC customer has no email, the export is skipped entirely with a logged reason.
- The `External Id` field on the company is set to the BC `Customer."No."` during export. On subsequent exports, the code checks for existing companies with the same External Id to prevent duplicates.
- Location addresses use `billingSameAsShipping: true` by default, meaning Shopify's billing and shipping addresses are always kept in sync.
- Contact roles are assigned during company creation based on the shop's `Default Contact Permission` setting. The enum values map directly to Shopify contact role names. `No Permission` skips role assignment entirely.
- The `Auto Create Catalog` shop setting triggers automatic catalog creation when a new company or company location is exported, connecting the B2B catalog to the new location.
- Company-level metafields are synced during the export phase via `ShpfyMetafields.SyncMetafieldsToShopify` if `Company Metafields To Shopify` is enabled on the shop.
- The tax registration ID mapping uses a pluggable interface (`ShpfyTaxRegistrationIdMapping.Interface.al`). The default implementation in `ShpfyTaxRegistrationNo.Codeunit.al` maps to `Customer."Registration Number"`, but the `ShpfyVATRegistrationNo.Codeunit.al` variant maps to `VAT Registration No.` instead. The shop setting `Shpfy Comp. Tax Id Mapping` controls which one is active.
