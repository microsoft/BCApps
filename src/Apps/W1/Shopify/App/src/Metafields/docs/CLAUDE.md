# Metafields

Metafields are Shopify's mechanism for attaching arbitrary typed key-value data to resources like products, variants, customers, and companies. This module provides a generic, polymorphic framework for reading, writing, and validating metafields across all supported owner types, bridging Shopify's flexible metadata system with BC's strongly-typed world.

## How it works

The module is built around two interface hierarchies that dispatch behavior based on enum values. The `Shpfy Metafield Type` enum (in `ShpfyMetafieldType.Enum.al`) implements `IMetafield Type`, and each of its ~25 values maps to a codeunit that knows how to validate, assist-edit, and provide example values for that type. For instance, `ShpfyMtfldTypeMoney.Codeunit.al` validates that the value is valid JSON with `amount` and `currency_code` keys, and opens a dedicated assist-edit dialog. Meanwhile, `ShpfyMtfldTypeBoolean.Codeunit.al` simply calls `Evaluate` and reports no assist-edit capability. The second axis is `Shpfy Metafield Owner Type` enum (in `ShpfyMetafieldOwnerType.Enum.al`), which implements `IMetafield Owner Type` -- each value (Customer, Product, ProductVariant, Company) maps to a codeunit that knows how to retrieve metafield IDs from Shopify via GraphQL, look up the Shop Code from the owner record, and determine whether metafields are editable given the shop's sync direction settings.

The single `Shpfy Metafield` table stores all metafields across all owner types. Records are keyed by the Shopify-assigned `Id` (BigInteger), and the `Parent Table No.` + `Owner Id` fields act as a polymorphic foreign key back to the owning record. When `Owner Type` is validated, the table trigger calls `IMetafieldOwnerType.GetTableId()` to auto-populate `Parent Table No.`, and vice versa. New metafields created locally receive negative IDs (decremented from -1) to distinguish them from Shopify-assigned ones until they are synced.

The API layer in `ShpfyMetafieldAPI.Codeunit.al` orchestrates sync in both directions. Export uses the `metafieldsSet` GraphQL mutation, which accepts a maximum of 25 metafields per call -- the code batches accordingly. It only sends metafields whose `Last Updated by BC` timestamp exceeds the Shopify `updatedAt` timestamp retrieved by the owner type's `RetrieveMetafieldIdsFromShopify`. Import parses the JSON response, skips values over 2048 characters (BC's field limit), silently drops unsupported types like `rating` and `rich_text_field`, and deletes local metafields that no longer exist in Shopify. The public-facing `ShpfyMetafields.Codeunit.al` provides the stable API surface for other modules.

## Things to know

- The `string` and `integer` enum values are legacy Shopify types. The table's `Type` field validation explicitly blocks them with an error directing users to `single_line_text_field` and `number_integer` respectively. The `CollectMetafieldsInBC` method also filters them out during export.
- The `money` type enforces that the currency code in the JSON value matches the shop's currency (or LCY if the shop has no currency set). This validation happens in the table's `Value` field trigger via `CheckShopCurrency`, not in the type's `IsValidValue` -- so the type codeunit validates structure while the table validates business rules.
- Metafield definitions can be imported from Shopify via `GetMetafieldDefinitions`, which creates empty metafield records with the correct namespace/key/type. Only the first 50 definitions are imported (single GraphQL page).
- The `Shpfy Metafield Value Type` enum is fully obsolete (tagged for removal in v28/29) and exists only behind `CLEANSCHEMA` preprocessor guards.
- Owner editability is context-dependent: product metafields require `Sync Item = To Shopify` AND `Can Update Shopify Products`; company metafields require `Can Update Shopify Companies` AND that company import is not set to `AllCompanies`. Each owner type codeunit encodes its own rules in `CanEditMetafields`.
- The assist-edit page (`ShpfyMetafieldAssistEdit.Page.al`) is a single multi-purpose dialog that shows/hides different field groups based on the type: money, dimension, volume, weight, and multi-line text each have dedicated sections. Simpler types like boolean, URL, and date have no assist-edit.
- All owner type codeunits follow the same GraphQL pattern to retrieve metafield IDs -- they query a specific `GraphQLType` (e.g., `ProductMetafieldIds`, `CompanyMetafieldIds`) and return a `Dictionary of [BigInteger, DateTime]` mapping metafield IDs to their `updatedAt` timestamps.
