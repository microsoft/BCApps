# Inventory

Inventory sync pushes stock levels from BC to Shopify -- BC is the source of truth. Shopify stock levels are imported only as a reference point to determine what has changed, and the sync calculates BC availability per location, then sets Shopify's inventory quantities to match. This module also manages the mapping between Shopify locations and BC locations, including fulfillment service locations.

## How it works

The sync process in `ShpfySyncInventory.Codeunit.al` runs in two phases. First, it imports current Shopify stock levels by iterating all shop locations that have a non-`Disabled` stock calculation. For each location, `ShpfyInventoryAPI.Codeunit.al`'s `ImportStock` fetches paginated inventory levels from Shopify's GraphQL API and upserts records in the `Shpfy Shop Inventory` table. Any inventory records not seen during this import (tracked via an `InventoryIds` list of SystemIds populated before the import begins) are deleted afterward by `RemoveUnusedInventoryIds` -- this cleans up products that have been removed from Shopify.

The export phase in `ExportStock` iterates all shop inventory records and calls `CalcStock` for each. This method first verifies the variant still exists and that the underlying BC item is not a service or non-inventory type. It then delegates stock calculation to the pluggable strategy via `GetStock`, which applies the configured `Stock Calculation` enum value for the location. The `Shpfy Stock Calculation` enum implements two interfaces simultaneously: `Shpfy Stock Calculation` for the actual calculation and `Shpfy IStock Available` to indicate whether the location should report stock at all. The built-in strategies are `Projected Available Balance Today` (uses `ItemAvailabilityFormsMgt.CalcAvailQuantities` in `ShpfyBalanceToday.Codeunit.al`) and `Non-reserved Inventory` (subtracts reserved qty from on-hand in `ShpfyFreeInventory.Codeunit.al`). The `Shpfy Extended Stock Calculation` interface extends the base with location-aware overload. After calculation, stock is adjusted for the variant's unit of measure if it differs from the base UOM. The final quantities are batched into GraphQL `inventorySetQuantities` mutations, 250 at a time, each with a unique idempotency key. Negative stock is clamped to zero. The `changeFromQuantity` is set to null (forcing a set rather than delta update).

Location management is handled by `ShpfySyncShopLocations.Codeunit.al`, which retrieves all locations from Shopify and reconciles them with the local `Shpfy Shop Location` table. Locations that disappear from Shopify are deleted locally (cascading to their inventory records). The sync also manages fulfillment service locations -- if a location named "Business Central Fulfillment Service" exists, the code updates its callback URL to the standard BC Shopify webhook endpoint.

## Things to know

- The `Shpfy Shop Inventory` table has a four-part composite key: `Shop Code` + `Product Id` + `Variant Id` + `Location Id`. The `Shopify Stock` field holds what was last imported from Shopify, while `Stock` holds what BC last calculated. The export only sends updates when these two differ.
- Stock is always rounded down to the nearest integer (`Round(..., 1, '<')`) before comparison and export. Fractional units from UOM conversion are truncated, not rounded.
- The `Shpfy Stock Calculation` enum is `Extensible = true`, so third-party extensions can add custom stock calculation strategies. The default implementation for `Disabled` returns zero and reports that the location cannot have stock.
- `Default Product Location` on `Shpfy Shop Location` controls whether new products in Shopify automatically get inventory at that location. You cannot mix standard locations and fulfillment service locations in this setting -- the validation prevents it.
- `Is Primary` on a shop location reflects Shopify's primary location designation and is read-only. `Default Product Location` is a separate BC-side setting.
- The `Location Filter` field on `Shpfy Shop Location` is a BC location code filter expression (supports wildcards), not a single location. When set to a single location, the `Default Location Code` is auto-populated. The `Default Location Code` is used on sales documents, while `Location Filter` controls which BC locations contribute to the stock calculation.
- The GraphQL inventory update uses `changeFromQuantity: null`, which tells Shopify to set the absolute quantity regardless of the current value. This avoids race conditions from concurrent updates but means the BC value always wins. The code includes retry logic (up to 3 attempts) for `IDEMPOTENCY_CONCURRENT_REQUEST` and `CHANGE_FROM_QUANTITY_STALE` errors.
- The `OnAfterCalculationStock` event in `ShpfyInventoryEvents.Codeunit.al` fires after stock is calculated, allowing subscribers to adjust the final number before it is recorded and exported.
